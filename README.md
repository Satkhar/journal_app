# Journal App

**Journal App** — desktop-приложение для журнала посещений и учёта турниров.
Проект написан на C++17/Qt6; локальные данные хранятся в SQLite.

### Описание

Приложение **Journal App** ведёт ежемесячный учёт посещений, карточки участников
и отдельный журнал турниров. Основной журнал и турниры сохраняются в
разных локальных SQLite-файлах.

Данные журнала за разные годы не разделяются по файлам: приложение ведёт одну
логическую базу клуба, а год и месяц хранятся в самих записях. Ручной выбор
«рабочей БД» и автоматическая смена файла по году не являются поддерживаемым
режимом. Причины и направление перехода к общей клубной системе зафиксированы
в [стратегии хранения данных](doc/data_storage_strategy.md).

### Функциональность

Основные возможности приложения:
- **Сохранение месяца**: в обычном загруженном месяце отдельная отметка
  посещения записывается сразу. После добавления состава из другого месяца
  изменения остаются черновиком и записываются только через
  `Данные -> Сохранить месяц`.
- **Чтение локальной базы**: `Данные -> Перечитать локальную базу` переключает
  приложение в `LOCAL` и показывает локальное состояние.
- **Добавление участника**: `Месяц -> Добавить участника` запрашивает обычное
  имя или ФИО и добавляет человека в текущий месяц. Историчное имя можно
  заполнить позже в карточке.
- **Удаление участника**: `Месяц -> Убрать выбранного участника` удаляет из
  текущего месяца участника, выбранного в таблице.
- **Добавление из другого месяца**: источник выбирается из выпадающего списка
  сформированных месяцев. Новые участники добавляются по UUID; текущий состав,
  посещения и отметки не удаляются. Результат остаётся черновиком до команды
  `Данные -> Сохранить месяц`.
- **Карточка участника**: необязательное историчное имя, ФИО, звание, боевая
  рука, контакт, ручной месяц начала тренировок, дата вступления в клуб,
  несколько пройденных званий с независимыми optional датами, заметка и один
  локальный герб. Отсутствующий флаг звания означает, что человек его не
  проходил; включённый флаг без даты означает неизвестную дату получения.
  Изображение герба нормализуется в PNG и сохраняется вместе с профилем одной
  транзакцией. Основная таблица показывает историчное имя, если оно заполнено,
  иначе ФИО. Участники группируются по текущему званию и сортируются по имени
  внутри каждой группы.
- **Статистика участника**: общее число посещений, первая и последняя
  записанные тренировки, среднее за завершённый месяц состава, тренировки в
  доспехах и проведённые тренировки, примерный стаж по вручную указанному
  месяцу начала, клубные турниры, выездные соревнования, СМБ-турниры, поездки
  без участия и бои. Месячная сетка показывает
  посещения и открывает выбранный месяц по нажатию.
- **Удары на время**: отдельная история замеров с рукой, количеством ударов,
  длительностью, мечом или тямбарой и примечанием. Скорость вычисляется из
  исходного замера; прогресс сравнивается только при одинаковых руке, оружии и
  длительности.
- **Возврат к текущей дате**: кнопка `Сегодня` в панели месяца использует тот
  же безопасный переход, что календарь, и не обходит защиту черновика.
- **Заголовок посещений**: дата и день недели находятся в закреплённом
  двухстрочном заголовке и остаются видимыми при прокрутке участников.
- **Сворачиваемый календарь**: компактная панель с месяцем и стрелками
  остаётся видимой; при запуске календарь по умолчанию свёрнут. Сформированный
  месяц можно сразу выбрать из выпадающего списка; стрелки позволяют открыть
  ещё не созданный месяц.
- **Отметки даты**: оплата, тренировка в доспехах, первое посещение, другое и
  признак того, что участник вёл тренировку в конкретный день.
- **Турниры и соревнования**: отдельный список с явным типом — клубный турнир
  на тренировке, выездное соревнование или СМБ-турнир (мягкий), — датой,
  бойцами клуба, сопровождающими, боями, счётом, внешними соперниками и общей
  заметкой. Человек из списка «Ездили, но не соревновались» получает отдельный
  счётчик поездок и не увеличивает число его турниров.

### Требования

Для работы с проектом вам потребуется:
- **CMake** (версия 3.16 или выше)
- **C++ компилятор** (совместимый с C++17)
- **MSYS2 MINGW64** с GCC/MinGW для Windows или GCC для Linux/macOS
- **Qt6** (`Core`, `Gui`, `Widgets`, `Sql`, `Network`)


для ubuntu

sudo apt install qt6-base-dev

### Установка

1. **Клонирование репозитория**
   ```bash
   git clone https://github.com/Satkhar/journal_app.git
   cd journal_app
   ```

2. **Создание директории сборки**
   ```bash
   mkdir build
   cd build
   ```

3. **Генерация файлов сборки**
   - Для Windows используйте терминал `MSYS2 MINGW64`; `cmake`, `gcc` и `Qt6`
     должны быть из `C:/msys64/mingw64`.
   - Для Unix-систем (Linux/macOS):
     ```bash
     cmake ..
     ```
   - Для Windows (`MSYS2 MINGW64`):
     ```bash
     cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
     ```

4. **Сборка проекта**
   - Для Unix-систем:
     ```bash
     make
     ```
   - Для Windows:
     ```bash
     cmake --build build -j
     ```

5. **Запуск приложения**
   После успешной сборки запустите исполняемый файл:
   - На Unix-системах:
     ```bash
     ./journal_app
     ```
   - На Windows:
     ```bash
     journal_app.exe
     ```

### Использование

#### Пример использования
1. Создание нового журнала:
   ```
   Запустить приложение.
   ```
   Для пустой базы или нового месяца приложение предложит создать месяц с нуля,
   перенести пользователей из другого месяца либо отложить настройку.

2. Добавление своих имен в журнал:
   ```
   Запустить приложение.
   При необходимости выбрать дни учета в меню создания месяца.
   Верхние переключатели `Пн`...`Вс` выбирают или снимают все даты
   соответствующего дня недели.
   При добавлении участников из другого месяца опция дней учёта применяет
   шаблон дней недели (пн, вт, ...), а не номера дат месяца.
   Выбрать `Месяц -> Добавить участника` и ввести обычное имя или ФИО.
   ..повторить..
   При необходимости отметить посещение выставление соотв. галочки.
   Колонка `Посещено` показывает число отмеченных активных дней участника и
   пересчитывается сразу после изменения отметки.
   Участники разделяются по текущему званию и сортируются по отображаемому
   имени внутри группы. Звание остаётся отдельной колонкой и визуальным
   признаком. UUID скрыт из основной таблицы и доступен только в подробной
   карточке участника.
   Кнопка `+` рядом с посещением открывает отметку дня. Доступны несколько
   одновременных событий: оплата, тренировка в доспехах, первое посещение,
   другое, вёл тренировку; также можно добавить комментарий. Цвет и значок
   показывают смысл отметки, подробности доступны во всплывающей подсказке.
   Выбрать `Данные -> Сохранить месяц`.
   ```
   Указанные имена будут сохранены в журнал с учетом отмеченных песещений.

3. Просмотр сохраненного журнала:
   ```
   Запустить приложение
   ```
   При старте приложения автоматически считывается сохраненный журнал и данные выводятся в таблицу.

4. Учёт турниров:
   - открыть `Справочники -> Турниры`;
   - выбрать клубный турнир на тренировке, выездное соревнование или
     СМБ-турнир;
   - отдельно указать соревновавшихся и ездивших без участия;
   - добавить строки боёв, выбирая нашего участника из списка или вводя имя
     внешнего человека вручную;
   - указать счёт и свободную общую заметку: места, выводы и задачи для разбора
     на тренировке.

5. Локальный server PoC:
   - следовать [инструкции Docker/libSQL](doc/docker_setup.md);
   - использовать только `127.0.0.1` и тестовые данные;
   - первый bootstrap schema выполнять явным environment flag из инструкции;
   - не открывать сервер в LAN/интернет: пока нет TLS, server-side auth и
     защиты month revision/CAS от конкурентной перезаписи.

Для новой установки `test_data.db` и отдельный `events_data.db` создаются в
системном каталоге локальных данных приложения. Если старый `test_data.db` уже
существует в рабочем каталоге, приложение продолжает использовать его и кладёт
`events_data.db` рядом, без неявного переноса пользовательских данных.
Абсолютный путь БД турниров показан в подсказке действия меню `Турниры`.
