# Journal App

**Journal App** — desktop-приложение для журнала посещений и учёта турниров.
Проект написан на C++17/Qt6; локальные данные хранятся в SQLite.

### Описание

Приложение **Journal App** ведёт ежемесячный учёт посещений, карточки участников
и отдельный журнал турниров. Основной журнал и турниры сохраняются в
разных локальных SQLite-файлах.

---

### Функциональность

Основные возможности приложения:
- **Сохранение месяца**: `Данные -> Сохранить месяц` записывает участников и
  посещения текущего месяца.
- **Чтение локальной базы**: `Данные -> Перечитать локальную базу` показывает
  локальное состояние, не переключая активный storage.
- **Добавление участника**: `Месяц -> Добавить участника` запрашивает историчное
  имя и добавляет человека в текущий месяц.
- **Удаление участника**: `Месяц -> Убрать выбранного участника` удаляет из
  текущего месяца участника, выбранного в таблице.
- **Карточка участника**: историчное имя, ФИО, звание, контакт и заметка.
- **Отметки даты**: оплата, особенная тренировка, первое посещение, другое и
  признак того, что участник вёл тренировку в конкретный день.
- **Турниры**: отдельный список соревнований с датой, внутренними
  участниками, боями, счётом, внешними соперниками и общей заметкой.

### Установка

1. **Клонирование репозитория**
   ```bash
   git clone https://github.com/Satkhar/journal_app.git
   cd journal_app
   ```

2. **Создание директории сборки**
   ```bash
   mkdir build
   cd build
   ```

3. **Генерация файлов сборки**
   - Для Windows используйте терминал `MSYS2 MINGW64`; `cmake`, `gcc` и `Qt6`
     должны быть из `C:/msys64/mingw64`.
   - Для Unix-систем (Linux/macOS):
     ```bash
     cmake ..
     ```
   - Для Windows (`MSYS2 MINGW64`):
     ```bash
     cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
     ```

4. **Сборка проекта**
   - Для Unix-систем:
     ```bash
     make
     ```
   - Для Windows:
     ```bash
     cmake --build build -j
     ```

5. **Запуск приложения**
   После успешной сборки запустите исполняемый файл:
   - На Unix-системах:
     ```bash
     ./journal_app
     ```
   - На Windows:
     ```bash
     journal_app.exe
     ```

### Использование

#### Пример использования
1. Создание нового журнала:
   ```
   Запустить приложение.
   ```
   Для пустой базы или нового месяца приложение предложит создать месяц с нуля,
   перенести пользователей из другого месяца либо отложить настройку.

2. Добавление своих имен в журнал:
   ```
   Запустить приложение.
   При необходимости выбрать дни учета в меню создания месяца.
   Верхние переключатели `Пн`...`Вс` выбирают или снимают все даты
   соответствующего дня недели.
   При переносе участников опция переноса дней учёта переносит шаблон дней
   недели (пн, вт, ...), а не номера дат месяца.
   Выбрать `Месяц -> Добавить участника` и ввести историчное имя.
   ..повторить..
   При необходимости отметить посещение выставление соотв. галочки.
   Колонка `Посещено` показывает число отмеченных активных дней участника и
   пересчитывается сразу после изменения отметки.
   Участники сортируются и визуально разделяются по званию. UUID скрыт из
   основной таблицы и доступен только в подробной карточке участника.
   Кнопка `+` рядом с посещением открывает отметку дня. Доступны несколько
   одновременных событий: оплата, особенная тренировка, первое посещение,
   другое, вёл тренировку; также можно добавить комментарий. Цвет и значок
   показывают смысл отметки, подробности доступны во всплывающей подсказке.
   Выбрать `Данные -> Сохранить месяц`.
   ```
   Указанные имена будут сохранены в журнал с учетом отмеченных песещений.

3. Просмотр сохраненного журнала:
   ```
   Запустить приложение
   ```
   При старте приложения автоматически считывается сохраненный журнал и данные выводятся в таблицу.

4. Учёт турниров:
   - открыть `Справочники -> Турниры`;
   - указать название, дату и наших участников;
   - добавить строки боёв, выбирая нашего участника из списка или вводя имя
     внешнего человека вручную;
   - указать счёт и свободную общую заметку: места, секундирование, выводы и
     задачи для разбора на тренировке.

Для новой установки `test_data.db` и отдельный `events_data.db` создаются в
системном каталоге локальных данных приложения. Если старый `test_data.db` уже
существует в рабочем каталоге, приложение продолжает использовать его и кладёт
`events_data.db` рядом, без неявного переноса пользовательских данных.
Абсолютный путь БД турниров показан в подсказке действия меню `Турниры`.
