# Error And Notes Log

Этот файл для фиксации найденных ошибок, технических замечаний и идей по улучшению.

## Как заполнять
- Добавляй новую запись в начало списка, чтобы сверху были самые свежие пункты.
- Для каждой записи указывай файл и короткий сценарий воспроизведения.
- После исправления меняй `Статус` на `fixed` и добавляй ссылку на commit/PR (если есть).

## Шаблон записи

```md
### [YYYY-MM-DD] Короткий заголовок
- Файл: `path/to/file`
- Тип: bug | architecture | cleanup | docs | test
- Приоритет: high | medium | low
- Описание: ...
- Как воспроизвести: ...
- Ожидаемое поведение: ...
- Фактическое поведение: ...
- Статус: open | in_progress | fixed | wontfix
- Комментарий: ...
```
